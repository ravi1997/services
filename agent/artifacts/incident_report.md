# Incident Report Template

## Summary
- What happened?
- Impact (users/services)
- Start time / detection time
- End time / mitigation time

## Symptoms
- Errors observed (502/504, crashes, etc.)
- Key log lines (redacted)

## Root Cause
- Technical root cause
- Contributing factors

## Mitigation
- Actions taken
- What restored service

## Corrective Actions
- Code/config changes
- Tests added
- Monitoring improvements

## Prevention
- Runbook updates
- Alerts and dashboards
